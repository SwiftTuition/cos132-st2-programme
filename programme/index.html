Good day, good people

This is your seven-day run-up to ST2. It's self-paced. Three tracks run in parallel and you decide how hard to push on each.

Every day has a recommended core: one video to watch, one pattern to reproduce, one mock test to attempt. That's the minimum I'd like you to hit. New tests, videos and pracs unlock as the week progresses, so the bank grows under you. If a recommended item doesn't click, replay the video, redo the test, or message me.

How It Works

There are three tracks, running side by side across the seven days. You don't have to do all three every day. Pick what you need.

Code with Swift

Reproduction drills. In each short video I write one common pattern from scratch and explain every line as I go. You watch, close the video, then write the same program yourself from a blank file. Doing it with your own hands is what makes the pattern stick.

Pracs

Architectural challenges. Three pracs over the week, with each one harder than the one before it. GitHub Codespaces, multi-file C++. They drop on Days 1, 4 and 5, and they're all due by the end of Day 7.

Mock Tests

Exam-difficulty self-checks. Fifteen tests in a bank organised by unit. One recommended each day, the rest open whenever you want a fresh attempt.

Architecture progression

The Code with Swift videos move through three stages of how a C++ program is laid out, from simplest to most realistic. By the end of the week you'll have written every stage yourself.

  • Stage 1. One file, everything in main(). The starting point.
  • Stage 2. One file, but with function declarations above main() and definitions below. Multiple functions, still one file.
  • Stage 3. Minimum three-file architecture — .h + .cpp + main.cpp. Include guards, header inclusion, multi-file compilation.

The 8 plans from L17

Every video uses one or more of the eight coding patterns from Lecture 17. All eight are covered by the end of Day 4. Days 5 to 7 then revisit those same patterns in pointer, array and file code, so you can see they show up almost everywhere, not only inside loops.

A read known count B read count first C sentinel D validate input E count F accumulate G max / min H swap

Mock Test Bank

Fifteen tests, organised by unit. Open from the day they land. The daily "recommended" is the one I'd suggest you start with. If you feel solid, one per unit is enough. If you want more reps, do all three.

Why three per unit

If you do badly on a test and want to retry the same scope, redoing the same paper doesn't teach you much because you already remember the answers. That's why each unit has three different papers covering the same scope at the same difficulty. The shape of the test stays the same; the questions are new.

Logo is still examinable

A lecturer recently confirmed that ST2 can ask Logo→C++ conversion questions. That includes conversions involving loops, and conversions where your C++ answer is expected to use pointers or references — even though Logo itself has neither. Every bank below includes at least one Logo→C++ conversion question pitched at the right difficulty for that unit.

Code with Swift Library

Seven reproduction-drill videos. Watch one, close it, then write the same program from scratch. Each video is short and focused on a single pattern, with no extra story or architecture work around it.

How to use a Code with Swift video
  1. Watch all the way through once. Don't pause.
  2. Close the video. Open a fresh file.
  3. Write the program from scratch. If you get stuck, write what you can and note where.
  4. Compare. Re-watch only the section you missed.
  5. Write it again. The second pass is where the pattern locks in.
